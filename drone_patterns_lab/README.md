drone_patterns_lab/
  drones/
    api/
      server.py - ініціалізує веб-сервер FastAPI 
      endpoints.py - обробляє HTTP-запити користувача для запуску місій та перевірки їх статусу
    factory/
      mission_factory.py - використовує налаштування, щоб створити всі компоненти (дрон, середовище, стратегії) у готову до запуску місію
      config_loader.py - перетворює вхідні дані у структурований об'єкт налаштувань
    observer/
      event_bus.py - пошта, доставляючи ці повідомлення від середовища до дрона без їх прямого зв'язку.
      events.py -  визначає структуру повідомлень (подій), які передаються системою
    cor/
      base.py - дозволяє створювати ланцюжок обробників та передавати запити від одного до іншого, chane respons
      reroute_handler.py - Спроба вирішити проблему з перешкодами на маршруті шляхом зміни курсу дрона.
      adjust_altitude_handler.py - Реагує на погану видимість або погодні умови, змінюючи висоту польоту.
      swarm_reassign_handler.py - Симулює передачу завдання іншому дрону з рою у випадку технічного збою
      emergency_land_handler.py - ініціює аварійну посадку при критичних помилках, якщо інші методи не допомогли.
    strategy/
      base.py 
      wave_reaction.py - Забезпечує стабілізацію морського дрона у випадку виявлення високих хвиль.
      wind_reaction.py - Реалізує алгоритм поведінки дрона при сильному вітрі
      crack_reaction.py - Змінює поведінку дрона (наближення до об'єкта) при виявленні дефектів поверхні під час інспекції.
    bridge/
      base.py - описує обов'язкові методи руху (зліт, посадка, переміщення) для будь-якої платформи
      controller.py - приймає команди від місії та перенаправляючи їх конкретній реалізації платформи.
      air.py - для повітряного дрона, що симулює роботу пропелерів та польоту
      sea.py - для морського дрона, що симулює роботу двигунів та плавання.
      surface.py - для наземного ровера, що симулює роботу коліс та двигуна.
    template/
      base.py - Цей файл реалізує патерн Template Method, визначаючи фіксований алгоритм виконання місії (зліт, навігація, дія, посадка)
    missions/
      sea_exploration.py - тут всі місії дрона 
      agriculture.py
      defects_detection.py
      rescue.py
      pollution_monitoring.py
    environment/
      base.py - тут всі середовища які імітуютья різні події
      air_env.py
      sea_env.py
      surface_env.py
    config/
      mission_config.py - dataclass,  кожна місія отримує повний набір необхідних параметрів (ID, тип, координати)
    utils/
      logger.py - логування
      telemetry.py - Відповідає за збір, накопичення та структурування даних (метрик) про стан дрона під час виконання місії.
      persistence.py - Реалізує механізм збереження фінальних результатів місії у файл
      math_utils.py - Містить допоміжний клас Coordinate для математичних операцій з позиціонуванням у просторі.
